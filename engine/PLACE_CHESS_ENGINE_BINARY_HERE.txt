Put your compiled engine binary here:

  Windows:        chess.exe
  macOS / Linux:  chess     (no extension; must have the execute bit set —
                              run `chmod +x chess` after copying it in)

The extension resolves the engine path in this order:

  1. The "chanakya.enginePath" setting in VS Code Settings, if you've set one.
  2. This folder: engine/chess.exe (Windows) or engine/chess (macOS/Linux).

Your engine must speak the UCI protocol (uci / isready / position / go /
bestmove) over stdin/stdout — this is standard for chess engines and is
what UCIEngine.js in this folder talks to.

If you already have your own engine/UCIEngine.js and engine/EnginePool.js
implementation from your original backend/src/ folder, feel free to replace
the versions in this extension with your own — EnginePool.js just needs to
expose: getEngine(key), requestMove(key, payload), newGame(), stopAll(),
disposeAll(). See extension.js for how they're called.

Delete this file once you've added the real binary.
